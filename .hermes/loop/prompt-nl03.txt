The approval queue has an endpoint like POST /api/applications/{id}/review?action=approve or POST /api/applications/{id}/review?action=reject. Currently it probably returns mock data or doesn't actually transition state in the database.

Wire the review endpoint to actually do these things:

1. POST /api/applications/{id}/review with action=approve:
   - Fetch the application from the DB
   - Verify it's in PENDING_REVIEW state
   - Transition: PENDING_REVIEW → STAGED
   - Create ApplicationEvent for the transition
   - Return 200 with updated application data

2. POST /api/applications/{id}/review with action=reject:
   - Fetch the application from the DB
   - Verify it's in PENDING_REVIEW state
   - Transition: PENDING_REVIEW → REJECTED
   - Create ApplicationEvent for the transition
   - Return 200 with updated application data

3. POST /api/applications/{id}/review with action=reset:
   - Transition from REJECTED back to PENDING_REVIEW
   - Same pattern

Error handling:
- Return 404 if application not found
- Return 409 if application is not in PENDING_REVIEW state
- Return 400 if action is invalid
- Wrap in try/except for DB operational errors -> 500
