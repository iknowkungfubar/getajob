Read /home/turin/getajob/getajob/cli.py to understand the existing command structure, and /home/turin/getajob/agents/tailoring_agent.py to understand how profiles are loaded.

Then add a new `profile` command group to the Typer app with these subcommands:

1. `getajob profile show` — Shows the currently active profile
   - Load profile from the database using ProfileStore
   - Display: name, email, location, skills (comma-separated), work_authorization
   - Use rich.table.Table for formatted output
   - If no profile exists, print "No profile configured. Run 'getajob init' to create one."
   - Handle the case where the database isn't available gracefully

2. `getajob profile update` — Interactive prompt to update profile fields
   - Show current values with option to keep them (press Enter to skip)
   - Fields: name, email, location, skills (comma-separated), work_authorization
   - Update via ProfileStore.update_profile() or equivalent
   - Print confirmation with updated fields

IMPORTANT: Use lazy imports for ProfileStore and database (import inside the function body, not at module top level). This prevents import errors when the database isn't configured.

Use the sub-app pattern: typer.Typer(name="profile") + app.add_typer(profile_app)

After adding, run: python3 -m pytest tests/ -q --tb=short
Then run: python3 -m getajob.cli profile show --help
